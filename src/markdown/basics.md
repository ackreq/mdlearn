## دستور زبان پایه

تقریباً همه برنامه‌ها و ویرایشگرهایی که از مارک‌دون پشتیبانی می‌کنند، دستور زبان پایه‌ای را که در نسخه اصلی جان گروبر معرفی شده دنبال می‌کنند. البته بین مفسرهای مختلف مارک‌دون تفاوت‌ها و اختلافات کوچکی وجود دارد که در فصل بعد توضیح داده می‌شوند.

### عنوان‌ها (سرفصل ها)

برای ایجاد عنوان (Heading) کافی است قبل از متن یا کلمه مورد نظر، از علامت `#` استفاده کنید. تعداد این علامت‌ها می‌تواند بین ۱ تا ۶ باشد؛ هر چه تعداد `#` بیشتر باشد، اندازه عنوان کوچکتر خواهد بود.

برای نمونه، عنوان با یک `#` بزرگ‌ترین و عنوان با شش `#` کوچک‌ترین است:

```rtl
# عنوان سطح ۱
## عنوان سطح ۲
### عنوان سطح ۳
#### عنوان سطح ۴
##### عنوان سطح ۵
###### عنوان سطح ۶
```

روش دیگر برای ایجاد عنوان، این است که در خط زیر متن، به اندازه دلخواه علامت‌های `=` برای عنوان سطح ۱ و علامت‌های `-` برای عنوان سطح ۲ قرار دهید:

```rtl
عنوان سطح ۱
===========

عنوان سطح ۲
-----------
```

روش دوم فقط برای عنوان‌های سطح ۱ و ۲ قابل استفاده است. همچنین تعداد علامت‌های `=` یا `-` می‌تواند متغیر باشد.

```admonish tip title="راهکارهای استاندارد"
برنامه‌های مختلف در نحوه‌ی برخورد با نبود فاصله بین علامت‌های `#` و نام عنوان توافق ندارند. برای اینکه نوشته شما در همه برنامه‌ها درست نمایش داده شود، همیشه یک فاصله بین `#` و نام عنوان قرار دهید.

همچنین بهتر است قبل و بعد از عنوان، یک خط خالی بگذارید تا سازگاری بیشتری ایجاد شود.
```

### پاراگراف‌ها

برای ساخت یک پاراگراف، بین متن‌ها یک خط خالی بگذارید:

```rtl
این خط اول است

و این خط دوم است.
```

اگر پاراگراف داخل یک لیست نیست، آن را با فاصله یا تب (Tab) شروع نکنید.

### شکستن خط (Line Breaks)

برای رفتن به خط بعد و شکستن خط فعلی بدون شروع پاراگراف جدید، کافی است در انتهای خط **دو یا چند فاصله** بگذارید و سپس کلید اینتر را بزنید.

```rtl
این خط اول است
و این خط دوم است.
```

این روش که به آن _فاصله پایانی_ یا _trailing whitespace_ گفته می‌شود، در بیشتر ویرایشگرهای مارک‌دون پشتیبانی می‌شود، اما همیشه ایدئال نیست. دلیلش این است که دیدن این فاصله‌ها در ویرایشگر سخت است و بسیاری از افراد به‌طور ناخواسته یا عمدی بعد از هر جمله دو فاصله می‌گذارند.

به همین دلیل، گاهی بهتر است از روش‌های دیگر برای ایجاد شکست خط استفاده کنید. اگر ویرایشگر مارک‌دون شما از HTML پشتیبانی می‌کند، می‌توانید از تگ `<br>` استفاده کنید که کاملاً واضح و قابل‌تشخیص است:

```rtl
این خط اول است <br>
و این خط دوم است.
```

### پررنگ کردن (Bold)

برای **پررنگ کردن متن**، دو علامت ستاره `*` یا دو underscore `_` را قبل و بعد از کلمه یا عبارت مورد نظر قرار دهید:

```rtl
**این متن بولد است**
__این هم بولد است__
این **بخش** از جمله بولد است.
```

### مورب کردن (Italic)

برای _مورب کردن_ متن، یک علامت ستاره `*` یا یک underscore `_` را قبل و بعد از کلمه یا عبارت مورد نظر قرار دهید:

```rtl
_این متن مورب است_
_این هم مورب است_
این *بخش* از جمله مورب است.
```

### پررنگ و مورب همزمان (Bold + Italic)

برای **_پررنگ و مورب کردن متن_** به طور همزمان، سه علامت ستاره `*` یا سه خط زیر `_` را قبل و بعد از کلمه یا عبارت مورد نظر قرار دهید:

```rtl
***متن مهم***
___متن مهم___
__*متن مهم*__
**_متن مهم_**
متن ***خیلی*** مهم

```

```admonish tip title="راهکارهای استاندارد"
برنامه‌های Markdown در مورد استفاده از زیرخط `_` وسط یک جمله توافق ندارند.
برای جلوگیری از ناسازگاری، بهتر است برای پررنگ و مورب کردن بین جملات از ستاره `*` استفاده کنید.
```

### نقل قول (Blockquote)

برای ایجاد **نقل قول**، یک علامت `>` را در ابتدای پاراگراف قرار دهید:

```rtl
> این یک نقل قول است.
```

خروجی نمونه به صورت زیر خواهد بود:

> این یک نقل قول است.

یک نقل قول می‌تواند شامل چند پاراگراف باشد. برای این کار، در خطوط خالی بین پاراگراف‌ها نیز یک علامت `>` قرار دهید:

```rtl
> این پاراگراف اول است.
>
> و این پاراگراف دوم است.
```

```admonish tip title="راهکارهای استاندارد"
برای سازگاری بیشتر، قبل و بعد از **نقل‌قول‌ها** یک خط خالی قرار دهید.
```

### لیست‌های مرتب (Ordered Lists)

برای ایجاد یک لیست مرتب، شماره‌گذاری را قبل از هر آیتم قرار دهید و بعد از آن یک نقطه بگذارید. اعداد لازم نیست به ترتیب واقعی باشند، اما بهتر است لیست با عدد `1` شروع شود:

```rtl
1. آیتم اول
2. آیتم دوم
3. آیتم سوم
```

مارکدون به طور خودکار شماره‌ها را مرتب می‌کند، حتی اگر شماره‌ها را اشتباه وارد کنید:

```rtl
1. آیتم اول
1. آیتم دوم
1. آیتم سوم
```

نتیجه همان لیست مرتب خواهد بود:

1. آیتم اول
2. آیتم دوم
3. آیتم سوم

برای ایجاد لیست‌های تو در تو، آیتم‌های داخلی را با چهار فاصله یا یک تب (Tab) فاصله دهید:

```rtl
1. آیتم اول
2. آیتم دوم
    1. آیتم تو در تو اول
    2. آیتم تو در تو دوم
3. آیتم سوم
```

نتیجه بصورت زیر نمایش داده می‌شود:

1. آیتم اول
2. آیتم دوم
   1. آیتم تو در تو اول
   2. آیتم تو در تو دوم
3. آیتم سوم

### لیست‌های بدون ترتیب (Unordered Lists)

برای ایجاد لیست بدون ترتیب می‌توانید در ابتدای هر آیتم از خط تیره `-`، ستاره `*` یا علامت جمع `+` استفاده کنید:

```rtl
- آیتم اول
- آیتم دوم
- آیتم سوم
```

```rtl
* آیتم اول
* آیتم دوم
* آیتم سوم
```

```rtl
+ آیتم اول
+ آیتم دوم
+ آیتم سوم
```

خروجی همه‌ی این روش یکسان است و به شکل زیر نمایش داده می‌شود:

- آیتم اول
- آیتم دوم
- آیتم سوم

برای ایجاد لیست‌های تو در تو، مشابه لیست‌های مرتب آیتم‌های داخلی را با چهار فاصله یا یک تب (Tab) فاصله دهید:

```rtl
- آیتم اول
- آیتم دوم
    - آیتم تو در تو اول
    - آیتم تو در تو دوم
- آیتم سوم
```

خروجی حاصل به صورت زیر در می‌آید:

- آیتم اول
- آیتم دوم
  - آیتم تو در تو اول
  - آیتم تو در تو دوم
- آیتم سوم

```admonish tip title="راهکارهای استاندارد"
برای سازگاری، از ترکیب کردن چند نشانه در یک لیست خودداری کنید — یک نوع نشانه انتخاب کرده و تا انتهای لیست از همان استفاده کنید.
```

### ترکیب لیست‌ها

می‌توانید یک لیست بدون ترتیب را داخل یک لیست با ترتیب قرار دهید، یا برعکس:

```rtl
1. مورد اول
2. مورد دوم
    - آیتم تو در تو اول
    - آیتم تو در تو دوم
3. مورد سوم
```

خروجی رندر شده به این شکل خواهد بود:

1. مورد اول
2. مورد دوم
   - آیتم تو در تو اول
   - آیتم تو در تو دوم
3. مورد سوم

### کد

برای مشخص کردن یک کلمه یا عبارت به عنوان `کد`، آن را داخل بک‌تیک `` ` ``
قرار دهید.

```rtl
این `بخش` از جمله یک کد است.
```

خروجی رندر شده به این صورت خواهد بود:

این `بخش` از جمله یک کد است.

اگر کلمه یا عبارتی که می‌خواهید به‌عنوان کد نمایش دهید شامل یک یا چند بک‌تیک باشد، می‌توانید آن را با قرار دادن در بین دو بک‌تیک پشت سر هم نمایش دهید.

```rtl
``از `code` در فایل Markdown خود استفاده کنید.``
```

خروجی رندر شده به این صورت خواهد بود:

`` از `code` در فایل Markdown خود استفاده کنید. ``

### بلوک‌های کد

برای ایجاد یک بلوک کد، هر خط آن را حداقل چهار فاصله (space) یا یک تب (Tab) به داخل (indent) ببرید.

```
    def greet(name):
        print(f"Hello, {name}!")

    greet("World")
```

در نهایت خروجی به شکل زیر خواهد بود:

    def greet(name):
        print(f"Hello, {name}!")

    greet("World")

همچنین می‌توانید از بلوک‌های کد حصار‌دار (Fenced Code Blocks) استفاده کنید که در ادامه توضیح داده خواهند شد. در این روش کد ها هایلایت خواهند شد و متن را زیبا تر میکنند.

### نمایش تصاویر

برای نمایش تصویر در مارک‌دون، از علامت تعجب `!` قبل از نوشتن متن جایگزین (alt text) استفاده می‌کنیم، سپس متن جایگزین را داخل کروشه `[]` می‌نویسیم و بعد آدرس یا مسیر تصویر را داخل پرانتز `()` قرار می‌دهیم.

ساختار کلی:

```
![متن جایگزین](آدرس-مسیر-تصویر-در-سیستم)
```

**مثال:**

```markdown
![Tux the Mascot](../files/Markdown/tux.png)
```

و خروجی حاصل به شکل زیر خواهد بود:

<div style="text-align: right;">
  <img src="../files/Markdown/tux.png" alt="Tux the Mascot" title="Tux the Mascot"
    style="max-width: 25%; height: auto;">
</div>

### خط جداکننده

برای ایجاد یک خط افقی، کافی است سه علامت یا بیشتر از یکی از موارد آورده شده را در یک خط قرار دهید: ستاره (`*`)، خط تیره (`-`) یا زیرخط (`_`):

مثال‌ها:

```
***

----

________
```

خروجی رندر شده همه‌ی این سه حالت یکسان به نظر خواهد رسید:

---

```admonish tip title="راهکارهای استاندارد"
برای سازگاری بهتر، قبل و بعد از خط افقی یک خط خالی قرار دهید.
```

### ایجاد لینک

برای ایجاد یک لینک به صورت زیر عمل میکنیم:

```markdown
[Google](https://google.com)
```

متن لینک داخل کروشه `[ ]` قرار می‌گیرد (`Google`) و آدرس مقصد لینک در پرانتز `( )` نوشته می‌شود (`https://google.com`). نتیجه‌ی این دستور به شکل [Google](https://google.com) نمایش داده می‌شود و با کلیک روی متن لینک، به آدرس موردنظر هدایت خواهید شد.

### آدرس‌های اینترنتی (URL) و ایمیل

برای اینکه یک URL یا آدرس ایمیل را به لینک تبدیل کنید، آن را داخل علامت‌های کوچک‌تر و بزرگ‌تر (`< >`) قرار دهید.

مثال‌ها:

```
<https://eff.org>
<fake@example.com>
```

خروجی نمایش داده‌شده به این صورت خواهد بود و می‌توانید روی آن‌ها کلیک کنید، بر خلاف متن عادی که قابل لینک نیست:

<div dir="ltr">
<a href="https://eff.org">https://eff.org</a><br>
<a href="mailto:fake@example.com">fake@example.com</a>
</div>

### نمایش کاراکترهای ویژه (Escaping Characters)

برای اینکه یک کاراکتر که معمولاً در مارک‌دون برای قالب‌بندی استفاده می‌شود را به‌صورت معمولی نشان دهید، کافی است قبل از آن بک‌اسلش (`\`) قرار دهید.

برای مثال، علامت ستاره (`*`) برای ایجاد لیست بدون ترتیب استفاده می‌شود و ما نمی‌خواهیم یک لیست ایجاد کنیم، بلکه می‌خواهیم تنها از علامت ستاره به‌صورت معمولی استفاده کنیم:

```rtl
\* بدون بک‌اسلش، این علامت به‌عنوان یک مورد در لیست بدون ترتیب نمایش داده می‌شد.
```

خروجی به این شکل خواهد بود:

\* بدون بک‌اسلش، این علامت به‌عنوان یک آیتم در لیست بدون ترتیب نمایش داده می‌شد.
